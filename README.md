[![Chrome Extension](https://img.shields.io/badge/Chrome-Extension-0f766e)](#安装)

[![Version](https://img.shields.io/badge/version-2.0.6-0f766e)](manifest.json)

一个面向 BOSS 直聘网页端的本地 Chrome 扩展：批量扫描岗位列表，按你的求职方向做粗筛，自动收藏高匹配岗位，并把所有判断过程导出成可复查表格。

它适合这些场景：

- 你每天要翻很多相似岗位，不想重复点开、判断、记录。
- 你想按「岗位性质 + 目标方向 + 薪资 + 排除词」稳定筛掉低质量岗位。
- 你希望收藏动作保守可控，拿不准的岗位进入待复核，而不是乱点。
- 你需要把扫描结果复制到飞书表格、Excel 或后续智能体里继续精筛。

如果这个项目帮你节省了筛岗位时间，欢迎点 Star。Star 越多，我会优先补更多平台适配、规则模板和导出能力。

## 核心亮点

- **多关键词自动搜索**：按搜索词逐个搜索 BOSS 岗位列表，确认结果列表切换后才开始扫描，避免扫旧结果。
- **逐岗位详情确认**：点击左侧岗位后，必须确认右侧详情标题、岗位编号或详情链接已切换到当前岗位，才会评分和收藏。
- **LLM 岗位判断**：岗位详情确认后必须调用 DeepSeek 返回分数、动作和理由；本地规则只作为参考上下文。
- **保守自动收藏**：高分岗位才尝试收藏；边界岗位进入待复核；收藏失败会单独记录。
- **DeepSeek 生成配置**：填写 OpenAI 兼容 API Key 后，点击生成搜索词会调用大模型生成岗位标题词。
- **完整导出复查**：支持复制表格、JSON、待复核链接、收藏失败记录、运行日志和诊断日志。
- **安全节奏控制**：每日扫描上限、每日收藏上限、每扫 N 个休息，避免长时间连续操作。
- **可控自动打招呼**：默认关闭；手动开启后，只对已收藏且达到最低分的岗位发送你编辑好的招呼内容。

## 它不会做什么

- 不自动投递简历。
- 不读取聊天记录。
- 不调用 BOSS 隐藏接口。
- 不承诺绕过验证码、登录异常或平台风控。
- 默认不自动打招呼，必须手动开启，并设置招呼内容和最低分。

遇到验证码、账号提示、登录异常或风控提醒时，应立即点击「暂停」并人工处理。

## 安装

1. 下载或克隆本仓库。
2. 打开 Chrome：`chrome://extensions/`
3. 打开右上角「开发者模式」。
4. 点击「加载已解压的扩展程序」。
5. 选择本项目文件夹。
6. 打开 BOSS 直聘网页版岗位列表，右下角会出现「BOSS-Auto-Job-Extension」面板。

## 推荐流程

1. 在 BOSS 页面顶部先手动设置城市、经验、学历、公司规模等基础筛选。
2. 在扩展面板填写「岗位性质」，例如 `销售`、`运营`、`客户成功`、`售前`、`产品`、`技术`。
3. 填写「目标方向」，例如 `AI客服`、`跨境电商`、`教育SaaS`、`企业服务`。
4. 点击「生成搜索词」。
5. 人工删掉太宽或不想看的搜索词，补上你知道的岗位叫法。
6. 点击「生成加分词/排除词」。
7. 选择「多关键词」模式，点击「启动/继续」。
8. 扫描结束后复制表格，重点复查「待复核」和「收藏失败」。

## 扫描模式

### 当前列表

只扫描当前页面已经加载出的岗位，不自动切换搜索词。适合临时检查某一个搜索结果页。

### 多关键词

按关键词框里的内容逐个搜索并扫描搜索结果。每个关键词最多扫描「每词」设置的数量，完成后自动搜索下一个词。扩展会等待搜索框/URL 与岗位列表签名确认后再扫描；如果结果未确认，会暂停任务而不是继续扫旧列表。

扫描单个岗位时，扩展会先点击左侧岗位卡片，再读取右侧完整详情容器。只有当前岗位标题、岗位编号或详情链接被确认匹配后，才会进入评分、收藏和打招呼流程；如果右侧详情仍停留在旧岗位，会暂停并记录 `job_detail_not_confirmed`，避免用旧详情做假判断。

多关键词规则：

- 关键词支持每行一个，也支持用逗号分隔。
- 勾选「自动扩展关键词」后，会根据岗位性质和目标方向补充相近搜索词。
- 勾选「最后扫推荐流」后，会先扫描所有搜索词，最后再扫描推荐流。
- 点击「暂停」会暂停当前任务；本页内再次点击「启动/继续」会继续当前关键词。
- 如果刷新页面或重新启动任务，会从新一轮任务开始，避免沿用旧进度造成假扫描。
- 勾选「跳过已扫」时，历史岗位只会被跳过，不会计入本次扫描数和每词进度。

## DeepSeek 判断逻辑

DeepSeek 负责生成配置，也负责每个岗位的最终判断。没有保存 API Key 时不能启动扫描。

生成分两步：

- **生成搜索词**：点击「生成搜索词」会请求 DeepSeek，生成 12-18 个可直接放进 BOSS 搜索框的岗位标题词；没有 Key 或调用失败时不会改用本地规则。
- **生成加分词/排除词**：请求 DeepSeek 生成少量评分锚点和明显不想看的岗位类型；没有 Key 或调用失败时不会改用本地规则。

例如 `销售 + AI应用`：

- 搜索词可以是 `Agent销售`、`智能体销售`、`AI软件销售`、`AI SaaS销售`、`AI解决方案销售`。
- 加分词只保留少量锚点，例如 `AI应用`、`Agent`、`AI软件`、`解决方案销售`。
- 排除词放明确不想看的岗位类型，例如 `保险`、`贷款`、`营业员`、`招生`、`课程销售`。

排除词会做冲突清洗。如果目标方向是 `机器人`，扩展不会把 `机器人硬件`、`人形机器人` 这类目标相关词直接当成排除词。

## 隐私和边界

高级规则里可以填写 OpenAI 兼容接口的 API Key 和模型名。默认模型是 `deepseek-v4-flash`。

- 不填写 Key：不能启动扫描，也不会用本地规则替代 LLM 判断。
- 填写 Key：生成搜索词、规则词和逐岗位判断时，会通过扩展后台请求 `https://api.toporeduce.cn/chat/completions`。
- Key 会作为 `Authorization: Bearer ...` 请求头发送到上述接口。只有确认信任该接口时才应填写。
- Key 保存在扩展的 `chrome.storage.local` 中；旧版本保存在页面 `localStorage` 的 Key 会在首次加载时迁移并删除。

## 安全限制

- 收藏分：默认 60 分，达到该分数才尝试收藏。
- 当前列表最多扫：当前列表模式单次最多处理多少个岗位，默认 300。
- 每词：多关键词模式下每个关键词最多扫描多少个岗位，默认 100。
- 今日最多扫：当天最多处理多少个岗位，默认 500。填 `0` 表示不限。
- 今日最多藏：当天最多收藏多少个岗位，默认 80。填 `0` 表示不限。
- 每扫 N 个休息：默认每扫 25 个岗位，随机休息 180-360 秒。
- 自动打招呼默认关闭；开启后只在收藏成功、分数达到「最低分」且未对该岗位发过招呼时尝试发送。
- 发送前会校验招呼内容已写入；发送按钮必须和消息输入框在同一聊天窗口内；未确认发送成功不会计入今日招呼。

## 导出内容

扩展支持复制：

- 全量表格
- 全量 JSON
- 待复核表格
- 待复核 JSON
- 待复核链接
- 已收藏记录
- 收藏失败记录
- 当前页岗位链接
- 诊断日志

表格会包含分数、岗位、公司规模、命中关键词、扣分关键词、主要原因、决策日志、收藏状态和链接。

## 结果状态

- **已收藏**：达到收藏分，并确认收藏成功。
- **待复核**：分数在复核区间，或达到收藏线但受安全限制影响。
- **已排除**：低于复核线，或命中硬排除条件。
- **收藏失败**：达到收藏分，但未确认收藏成功。
- **错误**：页面结构变化、岗位卡片识别失败、右侧详情未切换或按钮识别失败。

默认分层：

- 60 分及以上：尝试自动收藏。
- 50 分到收藏分以下：进入待复核。
- 50 分以下：跳过或排除。

## 故障保护

- 搜索结果未确认时会暂停，并在诊断日志记录 `keyword_search_timeout`。
- 右侧岗位详情未确认切换时会暂停，并在运行日志记录 `job_detail_not_confirmed`，此时应刷新页面或复制运行日志排查。
- 检测到验证码、安全验证、登录异常、访问频繁等提示时会停止任务，请人工处理。
- 风控检测会排除助手面板自身文案，避免把面板里的“登录异常/验证”说明误判为 BOSS 页面异常。
- 面板会显示最近运行事件；卡住时可点「更多导出与维护」里的「复制运行日志」。
- 面板会按当前浏览器缩放和视口大小自动收窄、单列显示，避免底部按钮和配置区域重叠。
- 每次改版前建议备份项目目录；本仓库维护流程使用 `F:\项目代码备份` 保存版本备份。
