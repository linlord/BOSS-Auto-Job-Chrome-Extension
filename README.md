[![Chrome Extension](https://img.shields.io/badge/Chrome-Extension-0f766e)](#快速安装)
[![Manifest V3](https://img.shields.io/badge/Manifest-V3-2563eb)](manifest.json)
[![Local First](https://img.shields.io/badge/Local--first-Job%20Screening-14b8a6)](#隐私和安全边界)
[![AI Scoring](https://img.shields.io/badge/AI-1--100%20Scoring-7c3aed)](#大模型配置)
[![Version](https://img.shields.io/badge/version-2.0.6-0f766e)](manifest.json)

一个给 BOSS 直聘网页端使用的本地 Chrome 扩展，帮你把重复、费眼、容易漏看的岗位筛选流程自动化。

它会按照你填写的岗位性质、目标方向、薪资边界、加分词和排除词批量扫描岗位，逐条打开右侧详情，让大模型给出 1-100 分匹配度、判断理由和建议动作，再把高匹配岗位自动收藏，把边界岗位放进待复核，把所有判断过程导出成可追踪的数据。

如果你每天都要在 BOSS 上翻很多相似岗位，这个扩展的目标很简单：少点机械点击，多点有效判断。

## 适合谁

- 正在密集找工作，需要每天批量筛选岗位的人。
- 想按「岗位性质 + 行业方向 + 关键词规则」稳定筛岗位的人。
- 不想把收藏交给纯关键词命中，仍希望每个岗位经过大模型判断的人。
- 需要把扫描结果复制到飞书表格、Excel、Notion 或后续分析流程的人。
- 想保留判断日志，方便复盘为什么收藏、跳过或进入待复核的人。

典型例子：

- 找 `AI 应用销售`、`AI SaaS 销售`、`智能客服销售`。
- 找 `AI 测试工程师`、`Web3 测试工程师`、`远程测试岗位`。
- 找 `客户成功`、`售前解决方案`、`产品运营`、`增长运营`。
- 排除保险、贷款、电话销售、招生、门店导购等明显不想看的岗位。

## 核心能力

### 1. 多关键词自动搜索

在面板里填入多个搜索词后，扩展会按顺序在 BOSS 搜索框中切换关键词，并等待页面结果确认更新后再开始扫描。

支持：

- 每行一个关键词，或用逗号分隔。
- 每个关键词独立设置最大扫描数量。
- 最后追加扫描推荐流。
- 跳过历史已扫描岗位。
- 暂停后继续当前关键词。

### 2. 大模型生成搜索词

填写岗位性质和目标方向后，点击「生成搜索词」会调用第三方大模型生成更贴近 BOSS 岗位标题的搜索词。

例如：

```text
岗位性质：销售
目标方向：AI 应用
```

可能生成：

```text
AI应用销售
AI软件销售
AI SaaS销售
Agent销售
智能体销售
AI解决方案销售
大模型销售
智能客服销售
AI知识库销售
```

生成后建议人工删掉太宽、太偏或不想看的词，再启动扫描。

### 3. 大模型生成加分词和排除词

点击「生成加分词/排除词」后，扩展会根据当前搜索词、岗位性质和目标方向生成少量评分锚点。

加分词用于让匹配岗位更容易被识别，排除词用于拦截明显不想看的岗位类型。它不会把整串搜索词直接复制成加分词，也会尽量避免把目标方向相关词误放进排除词。

### 4. 逐岗位打开详情后再评分

扩展不会只看列表卡片就下判断。它会先点击左侧岗位卡片，再确认右侧详情已经切换到当前岗位。

只有当标题、岗位编号或详情链接确认匹配后，才会继续：

- 读取岗位详情。
- 调用大模型评分。
- 判断收藏、待复核或排除。
- 记录原因、命中词和动作日志。

如果右侧详情没有切换成功，任务会暂停并记录原因，避免用旧岗位详情误判新岗位。

### 5. 1-100 分大模型评分

每个岗位最终由大模型给出 1-100 的细分分数，不再局限于 55、60、70 这类粗略档位。

评分会结合：

- 岗位标题和岗位详情。
- 岗位性质。
- 目标方向。
- 薪资边界。
- 用户配置的加分词和排除词。
- 本地规则给出的参考上下文。

收藏线和复核线只影响动作建议，不限制大模型给分。

### 6. 保守自动收藏

达到收藏分且通过安全限制后，扩展才会尝试点击收藏。

收藏结果会被记录为：

- 已收藏。
- 待复核。
- 已排除。
- 收藏失败。
- 错误。

边界岗位不会乱点，会进入待复核列表。

### 7. 可选自动打招呼

自动打招呼默认关闭。手动开启后，也只会对已收藏且达到最低打招呼分数的岗位尝试发送你配置好的招呼语。

发送前会确认：

- 招呼内容已经写入输入框。
- 发送按钮和输入框处在同一个聊天窗口内。
- 没有把扩展面板里的文案误识别成聊天输入框。

### 8. 状态灯和过程提示

面板会显示任务状态：

- 灰灯：未启动。
- 绿灯：正在运行。
- 红灯：暂停、报错或需要人工处理。
- 黄灯：任务完成。

每个状态后面都有文字说明，方便你知道当前是在搜索、扫描、等待、暂停还是完成。

### 9. 完整导出和日志

扩展支持复制：

- 全量表格。
- 全量 JSON。
- 待复核表格。
- 待复核 JSON。
- 待复核链接。
- 已收藏记录。
- 收藏失败记录。
- 当前页岗位链接。
- 运行日志。
- 诊断日志。

当大模型返回异常、岗位详情未切换、收藏按钮识别失败、页面出现风控提示时，可以复制日志排查。

## 快速安装

1. 下载或克隆本仓库。
2. 打开 Chrome，进入 `chrome://extensions/`。
3. 打开右上角「开发者模式」。
4. 点击「加载已解压的扩展程序」。
5. 选择本项目文件夹。
6. 打开 BOSS 直聘网页端岗位列表页。
7. 页面右下角会出现 `BOSS-Auto-Job-Extension` 面板。

## 推荐使用流程

1. 在 BOSS 页面顶部先手动设置城市、经验、学历、薪资、公司规模等基础筛选。
2. 打开扩展面板，填写「岗位性质」。
3. 填写「目标方向」。
4. 在「高级规则」中填写第三方大模型 Key 并保存。
5. 点击「生成搜索词」。
6. 人工检查搜索词，删掉太宽或不想看的词。
7. 点击「生成加分词/排除词」。
8. 设置收藏分、每词扫描数量、今日扫描上限、今日收藏上限。
9. 选择「多关键词」模式。
10. 点击「启动/继续」。
11. 扫描过程中重点看状态提示、待复核和收藏失败记录。
12. 扫描结束后复制表格或 JSON 做二次筛选。

## 大模型配置

扩展当前通过后台脚本请求 OpenAI 兼容接口：

```text
https://api.toporeduce.cn/v1/chat/completions
```

默认模型：

```text
deepseek-v4-flash
```

需要在面板的「高级规则」中填写第三方大模型 Key。没有保存 Key 时，下面这些动作会被强提示拦截：

- 生成搜索词。
- 启动或继续扫描。
- 生成加分词/排除词。

这是刻意设计的：岗位判断必须经过大模型，不会在失败时偷偷改用本地规则继续跑。

### Key 保存在哪里

- 新版本保存到扩展自己的 `chrome.storage.local`。
- 旧版本如果曾经保存到页面 `localStorage`，加载时会迁移到 `chrome.storage.local` 并删除旧值。
- 输入框会以中间星号的形式展示已保存 Key，不会明文显示完整 Key。
- Key 不写入仓库文件，不会因为提交代码而上传到 GitHub。

## 评分和结果状态

默认分层：

- 达到收藏分：尝试自动收藏。
- 复核线到收藏分之间：进入待复核。
- 低于复核线：跳过或排除。

常见结果：

| 状态 | 含义 |
| --- | --- |
| 已收藏 | 达到收藏分，并确认收藏成功 |
| 待复核 | 分数处在边界区间，或达到收藏线但受安全限制影响 |
| 已排除 | 明显不匹配、低分或命中硬排除条件 |
| 收藏失败 | 达到收藏线，但点击后未确认收藏成功 |
| 错误 | 页面结构变化、详情未切换、按钮识别失败或大模型调用失败 |

## 扫描模式

### 当前列表

只扫描当前页面已经加载出来的岗位，不自动切换搜索词。适合临时检查某一个搜索结果页。

### 多关键词

按关键词列表逐个搜索并扫描结果。每个关键词最多扫描「每词」设置的数量，完成后自动切换下一个关键词。

多关键词模式会额外确认：

- 搜索框内容已经变成当前关键词。
- URL 或列表内容已经发生切换。
- 岗位详情确实切换到当前点击的岗位。

如果确认失败，会暂停并记录日志，而不是继续用旧页面结果做判断。

## 隐私和安全边界

这个扩展是本地 Chrome 扩展，主要逻辑运行在你的浏览器里。

它不会做这些事：

- 不自动投递简历。
- 不读取聊天记录。
- 不调用 BOSS 隐藏接口。
- 不承诺绕过验证码、登录异常或平台风控。
- 不在没有 Key 的情况下启动大模型岗位判断。
- 不把你的 Key 写入 README、代码或仓库提交。

它会做这些事：

- 读取当前 BOSS 岗位列表和岗位详情页内容。
- 根据你的配置调用第三方大模型接口。
- 在达到条件时点击收藏按钮。
- 在你明确开启后尝试自动打招呼。
- 把运行记录保存在浏览器本地，供导出和排查。

遇到验证码、账号提示、登录异常、访问频繁等情况时，建议立即点击暂停，人工处理后再继续。

## 安全限制

面板内可以设置：

- 收藏分。
- 当前列表最大扫描数。
- 多关键词模式每词最大扫描数。
- 今日最大扫描数。
- 今日最大收藏数。
- 每扫描 N 个岗位后的随机休息时间。
- 自动打招呼最低分。

建议初次使用时保守设置，例如每词先跑 100-200 个岗位，确认搜索词和评分策略稳定后再放大。

## 故障排查

### 提示缺少 Key

打开「高级规则」，填写第三方大模型 Key 并保存。保存后输入框会显示脱敏后的 Key。

### DeepSeek 返回为空或不是 JSON

复制运行日志和诊断日志，检查：

- Key 是否有效。
- 模型名称是否正确。
- 网络是否能访问接口。
- 接口是否返回了 HTML 页面或错误对象。

### 右侧详情未切换

扩展会暂停，避免用旧详情判断当前岗位。可以刷新页面，或复制运行日志排查当前页面结构是否变化。

### 点击按钮触发 CSP 报错

扩展已经避免执行 `javascript:` 链接，收藏、打招呼、搜索等点击会走安全点击逻辑。如果仍出现类似报错，建议复制诊断日志反馈。

### 搜索词突然变成其他文案

扩展会隔离自动打招呼输入框和搜索词输入框，并在恢复任务时清洗搜索词内容。出现异常时可以清空当前搜索词重新生成。

## 开发和验证

本项目是原生 Chrome 扩展，没有构建步骤。

语法检查：

```bash

node -e "JSON.parse(require('fs').readFileSync('manifest.json','utf8')); console.log('manifest ok')"
```


安装验证：

1. 在 `chrome://extensions/` 重新加载扩展。
2. 刷新 BOSS 岗位列表页。
3. 确认右下角面板出现。
4. 保存大模型 Key。
5. 生成搜索词。

## 使用建议

- 先用小批量验证搜索词质量，再扩大扫描范围。
- 搜索词越像真实岗位标题，扫描结果越好。
- 加分词要少而准，排除词只放明确不想看的岗位类型。
- 不确定的岗位先进入待复核，不要急着排除。
- 每次页面结构变化或结果异常时，先看运行日志。
